# v1.0.4 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v1.0.4 with multi-UV inspection/preview and a lightweight GitHub Release update checker.

**Architecture:** Keep the current Electron + single-page renderer structure. Add small main-process IPC handlers for update checks and external links, and add renderer-side helpers for UV-channel detection, labels, layout drawing, and temporary preview-channel switching. Avoid parsing every FBX for UV metadata; only inspect models that are already loaded or parsed.

**Tech Stack:** Electron, Three.js, vanilla JavaScript, Node.js regression scripts.
